1 ## Linux Driver Compilation Instructions
2 
3 Supported Linux distributions are:
4 - Ubuntu 12.04, 14.04, 15.04, 15.10
5 - Debian Jessie
6 - Fedora 22, 23
7 
8 To compile bareflank, first start by downloading the repo, and running the
9 setup-<platform>.sh script, which creates the cross compiler and sysroot
10 that will be used later.
11 
12 ```
13 cd ~/
14 git clone https://github.com/Bareflank/hypervisor.git
15 cd ~/hypervisor
16 
17 ./tools/scripts/setup-ubuntu.sh
18 ```
19 
20 Once you have a cross compiler setup, you need to build the main source code.
21 This can be done by doing the following:
22 
23 ```
24 make
25 make linux_load
26 ```
27 
28 To cleanup everything, run the following:
29 
30 ```
31 make linux_unload
32 make clean
33 ```

35 ## Usage Instructions
36 
37 To use the bareflank hypervisor, you will use the bareflank manager (bfm).
38 This can be done manually by executing:
39 
40 ```
41 pushd bfm/bin/native
42 sudo LD_LIBRARY_PATH=. ./bfm load vmm.modules
43 sudo LD_LIBRARY_PATH=. ./bfm start
44 sudo LD_LIBRARY_PATH=. ./bfm status
45 sudo LD_LIBRARY_PATH=. ./bfm dump
46 popd
47 ```
48 
49 To stop the hypervisor, run the following:
50 
51 ```
52 pushd bfm/bin/native
53 sudo LD_LIBRARY_PATH=. ./bfm stop
54 sudo LD_LIBRARY_PATH=. ./bfm unload
55 popd
56 ```
57 
58 On Linux, you can also run the following shortcuts from the repo's root:
59 
60 ```
61 make load
62 make start
63 make stop
64 make unload
65 make quick
66 make status
67 make dump
68 make loop NUM=<xxx>
69 ```
70 
71 When extending the hypervisor, you need to provide the list
72 of modules you want to load. If you do this by hand, simply
73 replace vmm.modules with your own. If however you use the
74 shortcuts, do the following:
75 
76 ```
77 make [load|quick] MODULES=<path to list>
78 ```
79 
80 or
81 
82 ```
83 export MODULES=<path to list>
84 make [load|quick]
85 ```
